数据库结构文档&和大数据迁移计划


基础库
    - user_info.user_info_n:
        - 用途：作为用户来源 活动相关等的中间表；便于以少量计算需求中的部分指标，以预先字段并计算的方式减少统计时的计算负担；
        - 迁移计划：

    - orders_log.orders_log_n:
        - 用途：作为补充首充 用户来源 用户注册日期 活动充值 等相关数据的充值日志；便于以少量计算需求中的部分指标，以预先字段并计算的方式减少统计时的计算负担；
        - 迁移计划：保留现有字段，并将分表合并成1张表；

    - log_block.action_log_YYYYQN_N:
        - 用途：分块分区的日志行数据；通过用年和季度分区的形式将部分关心的日志行指标分区存储。
        - 迁移计划：直接使用大数据现有的日志行数据；


指标库
    - market_read:
        - 用途：各种做完的表都怼这里；


检测库
    - syn_monitor:
        - 用途：监控数据更新和数据准确性；
